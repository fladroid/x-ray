Chapter V

Memory

What changes when wandering becomes learning

The wanderer from the previous chapter had no memory. Each step was the first step. It could reach the goal a thousand times and still not know how to find it again. There was no accumulation — only noise.

Now we add one thing: a table. For every cell in the maze, the agent stores a number — its estimate of how valuable it is to be there. At first, all values are zero. The agent still explores randomly. But each time it reaches the goal, something propagates backward through its recent path. The cells that led to the reward become slightly more valuable. The table remembers.

This is Q-learning. The Q stands for quality — the quality of being in a given state. Over many episodes, the table fills with accurate estimates. The agent stops wandering and starts navigating.

"What fires together, wires together." — Donald Hebb, The Organization of Behavior, 1949

The right panel shows the Q-table as a heatmap. Each cell's color represents its current best estimated value — how useful it is to be there. At the start, everything is cold. Watch what happens after the agent reaches the goal for the first time: a warm signal appears at ★ and begins to spread. Cells near the goal warm up first. Cells far away take longer.

This is the X-Ray view of learning — not the agent's behavior on the surface, but the knowledge structure forming underneath.

Notice the asymmetry: the agent does not plan. It does not reason about paths. It only consults its table at each step and asks: which direction has the highest estimated value? The intelligence is in the table — not in the agent. The table is the memory. The agent is just the body that carries it.

This is what makes Q-learning an X-Ray tool: it externalizes knowledge into a structure we can inspect. We are not guessing what the agent has learned. We can see it — cell by cell, value by value.

But there is a limit waiting ahead. The maze here has 81 cells. A real problem — say, a game of chess, or a robot navigating a building — has millions of possible states. The table would need a row for each one. At some point, the table becomes too large to exist. That is where the next chapter begins.
